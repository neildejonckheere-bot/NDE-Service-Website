Work locally with Live Server in VS Code, push to GitHub when satisfied with changes. Never push just to preview — Live Server handles that.

**Why:** Avoids unnecessary commits and lets Neil see changes instantly without waiting for GitHub Pages to deploy.

**How to apply:** Always suggest Live Server for previewing. Only suggest pushing when a meaningful chunk of work is done.

---

Neil likes to talk through layout/design decisions before any code is changed. He'll say "let's talk about it first."

**Why:** Avoids back-and-forth from wrong assumptions about what he wants.

**How to apply:** When a layout question comes up, present options and ask for preference before touching any files.

---

Give broad strokes first, then details on request. Neil is capable and doesn't need hand-holding on every step.

**Why:** He has some Git experience and learns by doing.

**How to apply:** Explain what will happen, let him drive, step in with specifics only when asked.
